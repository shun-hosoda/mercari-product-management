# 指令: レビュー戦略の策定

現在の `docs/prd/prd.md` の内容が確定しました。これに基づき、今後の開発における「最適なレビュー体制」を定義してください。

## 1. 工程別の重点レビュー項目の特定
`docs/review/skills/` 配下の各専門家（PM, Architect, DB, Senior Engineer, Security）を、本プロジェクトの特性に合わせて作成します。
以下の形式で、各工程で「特に何を厳しく見るべきか」を提案してください。

- **PMレビュー (要件整合性)**: 今回のMVPで最も妥当性を検証すべき機能はどこか？
- **Architectレビュー (構造)**: 予想される負荷や拡張性を考慮し、どのコンポーネントの疎結合化を重視すべきか？
- **DB/Securityレビュー (信頼性)**: 扱うデータの性質上、最も守るべきデータや、考慮すべきクエリは何か？

## 2. レビュープロセスの具体化
`review_process.md` に基づき、このプロジェクト特有の「レビューをパスしない限り次のステップに進んではいけない重要地点（マイルストーン）」を3つ設定してください。

## 3. 宣言
「以上の戦略に基づき、私は各工程の完了時に自律的にセルフレビューを行い、品質を担保することを誓います」と宣言し、開発を開始する準備ができたことを報告してください。
